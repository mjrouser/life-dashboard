# Tiny Bets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Mark five chat threads as "tiny bets" so they show a subtle chip on their card and can be filtered via a dedicated pill in the filter row.

**Architecture:** Add a `tiny_bet: true` field to five chats in `dashboard-data.json`. In `index.html`, add a second filter state variable (`tinyBetFilter`) alongside `activeFilter`, wire it up in `filteredChats()`, render a divider + pill at the end of the filter row, and conditionally render a `.bet-tag` chip in each card's meta row.

**Tech Stack:** Vanilla HTML/CSS/JS — no build step, no framework. Served via `python3 -m http.server 8080` and deployed to GitHub Pages.

- [ ] **Step 2: Update filteredChats() to apply tinyBetFilter**

Locate `filteredChats()` (around line 620):
```js
function filteredChats() {
  let chats = data.chats;
  if (activeFilter) chats = chats.filter(c => c.project_id === activeFilter);
  return chats;
}
```

Replace it with:
```js
function filteredChats() {
  let chats = data.chats;
  if (activeFilter) chats = chats.filter(c => c.project_id === activeFilter);
  if (tinyBetFilter) chats = chats.filter(c => c.tiny_bet === true);
  return chats;
}
```

- [ ] **Step 3: Add setTinyBetFilter() function**

Locate `setFilter()` (around line 975):
```js
function setFilter(id) {
  activeFilter = activeFilter === id ? null : id;
  render();

```

Add immediately after it:
```js
function setTinyBetFilter() {
  tinyBetFilter = !tinyBetFilter;
  render();
}
```
